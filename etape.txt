DEMANDE NOUVEAU TITRE
[DEMANDEUR]
  → Saisir les informations du demandeur (nom, prénom, date naissance, contact...)
  → Chercher dans la base par email ou numéro de téléphone
       • SI TROUVÉ  → récupérer son ID et continuer
       • SI ABSENT  → créer le demandeur et récupérer son ID
       ↓
[PASSEPORT]
  → Saisir le numéro de passeport
  → Chercher dans la base par numero_passeport
       • SI TROUVÉ  → récupérer son ID
                    → vérifier que le statut = Actif (code 1)
                    → si non actif → bloquer et alerter l'utilisateur
       • SI ABSENT  → créer le passeport
                    → statut initial = Actif (code 1)
                    → enregistrer dans Historique_Statut_Passeport
       ↓
[VISA TRANSFORMABLE]
  → Saisir le numéro de référence du visa transformable
  → Chercher dans la base par numero_reference
       • SI TROUVÉ  → récupérer son ID
                    → vérifier que la date_expiration n'est pas dépassée
                    → si expiré → bloquer et alerter l'utilisateur
       • SI ABSENT  → créer le visa transformable
                    → lier au demandeur et au passeport récupérés
       ↓
[DEMANDE]
  → Créer la demande
  → id_type_demande = 1 (Nouveau titre)
  → id_statut = 1 (Créée)
  → id_demande_parent = NULL (pas de parent)
  → lier au demandeur, passeport et visa transformable récupérés
       ↓
[PIÈCES JUSTIFICATIVES]
  → Récupérer automatiquement les pièces requises selon le type de visa
  → Attacher chaque pièce à la demande
  → est_fourni = FALSE par défaut (le demandeur les fournira après)
       ↓
[HISTORIQUE]
  → Enregistrer le statut initial = Créée (code 1)



TRANSFERT VISA
[DEMANDEUR]
  → Saisir les informations du demandeur
  → Chercher dans la base par email ou numéro de téléphone
       • SI TROUVÉ  → récupérer son ID et continuer
       • SI ABSENT  → créer le demandeur et récupérer son ID
       ↓
[ANCIEN PASSEPORT]
  → Saisir le numéro de l'ancien passeport
  → Chercher dans la base par numero_passeport
       • SI TROUVÉ  → récupérer son ID
                    → vérifier qu'il appartient bien à ce demandeur
       • SI ABSENT  → créer l'ancien passeport (venant de l'ancien système)
                    → statut = Actif (code 1)
                    → enregistrer dans Historique_Statut_Passeport
  → Récupérer son ID dans tous les cas
       ↓
[NOUVEAU PASSEPORT]
  → Saisir le numéro du nouveau passeport
  → Chercher dans la base par numero_passeport
       • SI TROUVÉ  → récupérer son ID
                    → vérifier que le statut = Actif (code 1)
                    → vérifier que ce n'est pas le même que l'ancien
       • SI ABSENT  → créer le nouveau passeport
                    → statut = Actif (code 1)
                    → enregistrer dans Historique_Statut_Passeport
  → L'ancien passeport reste dans la base (on ne le supprime pas)
       ↓
[DEMANDE PARENT]
  → Créer une demande de type Nouveau titre (type = 1)
  → id_statut = 1 (Créée)
  → id_demande_parent = NULL
  → Cette demande représente la base administrative du dossier
       ↓
[DEMANDE ENFANT]
  → Créer une demande de type Transfert VISA (type = 2)
  → id_statut = 1 (Créée)
  → id_demande_parent = ID de la demande parent créée juste avant
  → Cette demande porte la logique du transfert
       ↓
[DEMANDE_TRANSFERT]
  → Lier la demande enfant à :
       • id_ancien_passeport → ID récupéré ou créé (ancien système)
       • id_nouveau_passeport → ID récupéré ou créé
  → C'est ici qu'on sait d'où vers où le visa est transféré
       ↓
[PIÈCES JUSTIFICATIVES]
  → Récupérer automatiquement les pièces requises pour le transfert
  → Attacher sur la demande ENFANT
  → est_fourni = FALSE par défaut
       ↓
[HISTORIQUE]
  → Enregistrer statut Créée (code 1) pour la demande PARENT
  → Enregistrer statut Créée (code 1) pour la demande ENFANT



DUPLICATA CARTE RÉSIDENT
[DEMANDEUR]
  → Saisir les informations du demandeur
  → Chercher dans la base par email ou numéro de téléphone
       • SI TROUVÉ  → récupérer son ID et continuer
       • SI ABSENT  → créer le demandeur et récupérer son ID
       ↓
[PASSEPORT]
  → Saisir le numéro de passeport
  → Chercher dans la base par numero_passeport
       • SI TROUVÉ  → récupérer son ID
       • SI ABSENT  → créer le passeport et récupérer son ID
                    → statut initial = Actif (code 1)
                    → enregistrer dans Historique_Statut_Passeport
       ↓
[CARTE RÉSIDENTE ORIGINALE]
  → Saisir la référence de la carte résidente perdue/volée
  → Chercher dans la base par reference
       • SI TROUVÉE → récupérer son ID
                    → vérifier qu'elle appartient bien à ce demandeur
       • SI ABSENTE → la carte vient peut-être de l'ancien système
                    → saisir manuellement les informations de la carte
                         • reference
                         • date_debut
                         • date_fin
                         • id_passeport (récupéré ou créé juste avant)
                    → créer la carte dans carte_resident
                    → récupérer son ID
                    → continuer normalement
       ↓
[MISE À JOUR PASSEPORT]
  → Saisir le motif : Perdu / Volé / Détérioré

  TABLE : Passeport
    → UPDATE id_statut_actuel
         • Perdu      → code 21
         • Volé       → code 31
         • Détérioré  → pas de UPDATE (statut reste Actif = 1)

  TABLE : Historique_Statut_Passeport
    → INSERT du nouveau statut
         • Perdu      → enregistrer code 21
         • Volé       → enregistrer code 31
         • Détérioré  → pas d'INSERT
       ↓
[DEMANDE PARENT]
  → Créer une demande de type Nouveau titre (type = 1)
  → id_statut = 1 (Créée)
  → id_demande_parent = NULL
  → id_visa_transformable = NULL (pas de visa transformable ici)
  → Cette demande représente la base administrative du dossier
       ↓
[DEMANDE ENFANT]
  → Créer une demande de type Duplicata (type = 3)
  → id_statut = 1 (Créée)
  → id_demande_parent = ID de la demande parent créée juste avant
  → id_visa_transformable = NULL
       ↓
[DEMANDE_DUPLICATA]
  → Lier la demande enfant à :
       • id_carte_originale → ID récupéré ou créé (ancien système)
       • motif → 'Perdu' ou 'Volé' ou 'Détérioré'
  → C'est ici qu'on sait quelle carte on est en train de refaire
       ↓
[PIÈCES JUSTIFICATIVES]
  → Récupérer automatiquement les pièces requises pour le duplicata
  → Attacher sur la demande ENFANT
  → est_fourni = FALSE par défaut
       ↓
[HISTORIQUE]
  → Enregistrer statut Créée (code 1) pour la demande PARENT
  → Enregistrer statut Créée (code 1) pour la demande ENFANT
